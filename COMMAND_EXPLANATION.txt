================================================================================
          ASIC DESIGN IMPORT & INITIALIZATION SCRIPT README
================================================================================
Overview:
This document explains the "Importing Design" stage of the ASIC physical 
design flow. This script is responsible for setting up the initial database 
by loading the synthesized gate-level netlist, physical libraries (LEF), 
Multi-Mode Multi-Corner (MMMC) timing constraints, and IO pad placement files. 
It specifically targets a 180nm technology node with standard cells, IO pads, 
and memory macros.

================================================================================
1. INITIALIZATION VARIABLES CONFIGURATION
================================================================================
These TCL variables define the paths and settings required to build the initial 
database:

* `set design_mode 180`
  Sets the target process technology node to 180nm.

* `set init_design_netlisttype {Verilog}` / `set init_verilog ...`
  Specifies the format and path of the input gate-level netlist generated by 
  the logic synthesis tool.

* `set init_top_cell mac_4x4_wrapper_chip`
  Explicitly defines the top-level module name of the design hierarchy.

* `set init_lef_file {...}`
  Defines the list of Library Exchange Format (LEF) files. These files contain 
  the abstract physical representations (metal layers, pins, boundaries) of:
  - Technology rules (tsl180l6.lef)
  - Standard cells (scl18fs120_std.lef)
  - IO pads (tsl18cio150_6lm.lef)
  - A specific Dual-Port SRAM memory macro (rd3_1024x8.lef)

* `set init_io_file ./final.io`
  Provides the IO assignment file, which dictates exactly where the external 
  pins or IO pad cells should be placed along the die boundary.

* `set init_mmmc_file ./Default.view`
  Loads the MMMC setup file, which points to the timing libraries (.lib), 
  RC extraction corners, and SDC constraints required for Static Timing Analysis.

* `set init_pwr_net` / `set init_gnd_net`
  Explicitly defines the names of the global power (VDD_CORE, VDDO_CORE) and 
  ground (VSS_CORE, VSSO_CORE) nets to be used throughout the chip.

* `set floorplan_default_site CoreSite`
  Defines the standard cell site/row type for the floorplan.

================================================================================
2. DATABASE INITIALIZATION
================================================================================
* `save_global Default.globals`
  Saves all the `init_*` variables configured above into a reusable Cadence 
  ".globals" file for easier loading in future sessions.

* `set_message -no_limit`
  Removes the default cap on the number of printed warning and error messages in 
  the log file, ensuring no critical setup issues are hidden.

* `setDesignMode -process $design_mode`
  Applies the 180nm process variable to the tool's internal engine.

* `init_design`
  The core command of this script. It reads the netlist, LEF files, IO file, 
  and MMMC setup defined by the variables, compiles them, and builds the initial 
  physical design database in memory.

* `add_tracks -honor_pitch`
  Generates the routing grid overlay for the design based on the minimum pitch 
  and spacing rules defined in the technology LEF.

================================================================================
3. PRE-FLOORPLAN VERIFICATION & REPORTING
================================================================================
* `checkUnique -verbose`
  Checks the loaded netlist for any non-unique instances or multiple 
  instantiations of the same module that could cause database conflicts.

* `check_timing -verbose`
  Analyzes the loaded SDC constraints to ensure all registers have clocks, 
  and there are no unconstrained timing paths.

* `checkDesign -all`
  A comprehensive sanity check on the database, looking for missing library 
  cells, disconnected nets, or inconsistencies between the physical (LEF) and 
  logical (Netlist) domains.

* `reportGateCount -hinst ...`
  Reports the total number of standard cell gates and instances in the design, 
  broken down by hierarchy.

* `reportNetStat`
  Generates a statistical report on the electrical nets in the design (e.g., 
  fanout distributions, longest nets).

================================================================================
4. CHECKPOINTING
================================================================================
* `saveDesign ./Import_Design/..._importdesign.enc`
  Saves the complete, fully initialized and checked Innovus database to disk. 
  This acts as the baseline starting point before moving to the Floorplanning 
  stage.
